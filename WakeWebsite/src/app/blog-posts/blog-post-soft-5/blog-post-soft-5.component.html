Communication Between Project Components Part 3: Websockets Implemented and Tested on Mirror Frontend – February 22, 2021 Software Team

The magic mirror frontend was modified to use Websockets instead of ROS as stated in the previous software blog. Once this was done we had to make sure all the modules worked and were responsive to requests coming in on the Websocket. Additionally, we had to test what a happy path of the alarm loop would look like to make sure it worked well.

This was done by creating a Python script to mock all the possible requests that the mirror could get from the other components of the project. The video below shows the mirror being controlled by said Python script to test and demonstrate the mirror’s capabilities. In the video, the terminal on the left is the logs being outputted by the Python script and the terminal on the right shows the requests being received by the magic mirror frontend.
